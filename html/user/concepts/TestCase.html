TestCase

A TestCase is an XML file that conforms to the german BSI XML structure. It will be structured in PreCondition, TestSteps, and PostCondition.

Communication between the chip or simulator and reader or prover in PreCondition, TestSteps, and PostCondition will be organized in tags as described for the XML structure.

PreCondition

A PreCondition may be a state of the card or simulator that can be achieved by performing some operations before the TestSteps (that is the purpose of the TestCase) are performed.

TestStep

In TestSteps, the actual purpose TestCase will be implemented.

PostCondition

A PostCondition may be a state of the card or simulator that should be achieved after the TestSteps are performed.

XML structure

A TestCase in these XML tags:

  • Title: The Title of the TestCase. It should correspond to the title of the test case in its according test specification.
  • Version: The version number the test case has according to its test specification.
  • Purpose: A short description of the purpose of the test case.
  • Profile: A test specification often defines profiles. Each test case has some of them assigned to. They will be listed in this tag.
  • Reference: Reference can be listed in this tag, for example the technical guideline the test specification is based on.
  • PreCondition
  • TestSteps
  • PostConditions

PreCondition, TestSteps and PostConditions are organized in the following tags:

  • Command: A tag that gives the test step a title, i.e. "Reset Card".
  • Description: A short description of the purpose of the test step.
  • TechnicalCommand: In this tag the test step is executed. Commands and APDUs can be send and received.
  • ExpectedResult: In this tag the result APDU can be verified, i.e. its status words and response data (if any) can be checked. It is structured in two possible tags as well:
    • Text: Gives a short textual description of the expected result, i.e. "SW 9000" if the status word 9000 is expected.
    • TechnicalResult: Status words and response data can be checked in assertions.
