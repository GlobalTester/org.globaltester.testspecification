• Command: A tag that gives the test step a title, i.e. "Reset Card".
  • Description: A short description of the purpose of the test step.
  • TechnicalCommand: In this tag the test step is executed. Commands and APDUs can be send and received.
  • ExpectedResult: In this tag the result APDU can be verified, i.e. its status words and response data (if any) can be checked. It is structured in two possible tags as well:
    • Text: Gives a short textual description of the expected result, i.e. "SW 9000" if the status word 9000 is expected.
    • TechnicalResult: Status words and response data can be checked in assertions.
