Create and Export a TestSpecification

In a few steps, this user guide will show you how to create and export a TestSpecification. In order to get a quick overview, you can also use the Cheat Sheet.

Create your own TestSpecification

With GlobalTester 3 you cannot only use imported TestSpecifications, but you can also write your own. You can also share it with the GlobalTester Community if you like to.

First, go with the mouse courser in the "GlobalTester Navigator" window. You might already have some TestSpecifications or TestCampaigns created there. Right-click, and choose "New -> Project...".

In the "New Project" wizard, select "GlobalTester -> TestSpecification", and click "Next >". Enter a name for your TestSpecification Project in the text field "Project name". If the name typed in the field is already existing in your GlobalTester 3 runtime workspace, you will be notified.

Then click "Finish"

Add a TestCase

In the "GlobalTester Navigator" window, your own TestSpecification will be shown as a project folder. In its sub-folder TestCases, you can now add a default TestCase that will select the master file:

Go with the mouse courser in the "GlobalTester Navigator" window and choose the "TestCases" folder. Right-click, and choose "New -> Other...".

In the "New" wizard, select "GlobalTester -> TestCase", and click "Next >". Enter a name for your TestCase in the text field "File name". Then click "Finish"

Write a TestCase

You can use this default TestCase as a basis for your own TestCase. It already comprises the XML structure it is supposed to have in order to be executable. You can find an example of how to send an APDU command and evaluate its response in it, i.e. status word and response data.

For further and specific information, have a look at the User Guide "Write TestCases", and also the XML structure for TestCases.

Add TestCase Data

If you need any own card-verifiable certificates, you can specify them in the "Test Data -> Certificates" folder using the XML structure for Certificates.

You can create and your own Subroutines and use them in you TestCase(s) as well.
