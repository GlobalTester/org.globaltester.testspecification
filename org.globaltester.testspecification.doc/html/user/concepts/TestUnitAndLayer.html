TestUnits and TestLayers

TestUnits and Layers are XML-structures, which are used to group different collections of TestCases. This XML-Structures are an enhanced Version of the formerly used TestSuites (which are still supported by the GlobalTester). TestUnits are used to reference on TestCase files while TestLayers reference on TestUnit files. This is normally used to test on different layers within one test session.

The structure of these files is quite simple. A TestUnit (root tag TestUnit) contains a Text tag and (one or more) TestCase tags which contains the path to the test files. The Text tag also contains a Paragraph Tag with an description of the Unit.

TestLayers are implemented in a very similar way. A TestLayer (root tag TestLayer) also contains a Text tag. This tag has a Heading tag to describe the Layer test. Instead of directly referencing TestCases it contains one or more TestUnits tags which reference to the previously mentioned TestUnit files.
