XML Structure for Certificates

A card verifiable certificate is structured in these XML tags:

  • SignerCertificate (optional): If the certificate has a (regular) signer, its certificate is linked.
  • CertificateAuthorityReference (optional, if a SignerCertificate is linked): A string that specifies the Certificate Holder Reference of the signer certificate.
  • SigningKey (optional, if a SignerCertificate is linked): The public key to verify the certificate.
  • CertificateHolderReference: A string that identifies the public key contained in the certificate.
  • CertificateHolderAuthorization: A string that lists the access rights that will be encoded in the Certificate Holder Authorization Template byte(s).
  • CertificateEffectiveDate: The effective date of the certificate.
  • CertificateExpirationDate: The expiration date of the certificate.
  • PublicKey: The public key of this certificate.
  • CertificateExtension (optional): They can encode the Discretionary Data Template that will encode the Terminal Sectors for Restricted Identification.
